Pelican系列:生成博客，预览博客，将博客发布到GitHub，绑定域名

Date 2016-03-29 Tags Pelican

网站生成

文章写好了，接下来当然是要将写好的文章转化为HTML文件了，Pelican提供了一些命令来帮助我们 生成HTML文件

pelican /path/to/your/content/ [-s path/to/your/settings.py]

执行上面这条命令之后，生成的HTML代码会被输出到output/下面，这个目录也可以在配置文件 里面进行修改，如果没有配置主题的话，会使用默认的主题。

如果希望Pelican监听文章的变化，即每次保存文章内容就自动生成了HTML文件，可以在命令里面添 加参数-r或者--autoreload

关于其他Pelican详细的命令说明可以查看Pelican的帮助

pelican --help

预览博客

第一种方式就是直接用双击index.html文件，用浏览器打开，还有一种方法当然就是起一个小小服 务器。

如果是Python 2

cd output
python -m SimpleHTTPServer

如果是Python 3

cd output
python -m http.server

服务启动之后在浏览器输入地址:http://localhost:8000/就可以看看博客的整体效果了。

将博客发布到GitHub上面

首先肯定是需要一个GitHub账户，关于GitHub的教程网上很多，可以找一下，这里我主要说怎么 利用GitHub部署我们的博客。还有一个git的基本用法，这里就不再详细说了，网上的教程也是 很多。

一般我会创建两个项目，一个用来管理生成的文件，一个用来管理自己写的内容。我们先创建用 来发布博客的项目，项目名称必须是：用户名.github.io，比如我的是xinxingzhao.github.io， 然后执行下面命令，将我们生成的html文件发布到github上面。

初始化项目

cd output
git init
git add .
git commit -m "init commit"

关联远程仓库，如果不喜欢在output作为项目管理的名称，在将输出目录改为: output/xinxingzhao.github.io，这样本地管理的项目名称就和github上面的一样了

git remote add origin git@github.com:xinxingzhao/xinxingzhao.github.io.git

将博客发布到远程仓库

git push origin master

现在就可以通过http:/xinxingzhao.github.id进行访问博客了，是不是很酷啊！

另外我们还要管理我们写的博客，就是除了output之外的几个目录和文件，我们继续在 github上面创建一个项目：blog(这个项目成功不做限制，可以起个你喜欢的名字)。

在本地初始化项目的时候，和上面那个项目是不一样的，因为output里面是另外一个项目 在管理了，我们就不需要再继续管理里面的文件，我们就可以添加。gitignore文件, 忽略掉不需要管理的目录。

cd ../
touch .gitignore
echo 'output/' >> .gitignore
git init
git add .
git commit -m "init commit"
git remote add origin git@github.com:xinxingzhao/blog.git
git push origin master

域名绑

可能这个时候还不够爽，如果绑定一个自己的域名那个更加的高大上的，装逼指数直线上升。

首先需要申请一个域名，可以到万网上面申请一个，第一年都很便宜，也就几十块钱，这个 应该是唯一需要投入的地方吧，空间就不要买了
