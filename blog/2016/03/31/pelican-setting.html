Pelican系列:个性化设置

Date 2016-03-31 Tags Pelican

这部分的详细内容比较多，一点一点慢慢的更新了。

通过下面的命令可以生成配置文件

pelican content -s path/to/your/pelicanconf.py

如果是使用pelican-quickstart创建博客骨架的话，配置文件会自动生成

温馨提示：如果在实验不同的配置的时候，特别是元数据的配置，可能因为缓存 的原因而没有发生变化，可以通过配置LOAD_CONTENT_CACHE = False或者 在命令中添加参数--ignore-cache处理缓存的问题

配置文件pelicanconf.py所有等号前面的必须是大写的，否则Pelican将 无法解析配置文件，除了Number，boolean，dictionaries和tuples不需要引号 因起来，其他类型必须用引号引起来。

基础配置

  • AUTHOR : 默认文章作者，可以给自己起个笔名什么的；

  • DATE_FORMATS = {} : 如果博客是多种语言的，可以为每种语言设置一种日期格式；

  • USE_FOLDER_AS_CATEGORY = True : 如果在文章里面没有设置Category属性， 那么将取文章所在的子目录作为分类，如果USE_FOLDER_AS_CATEGORY设置为False， DEFAULT_CATEGORY将作为默认分类；

  • DEFAULT_CATEGORY = 'misc' : 默认分类；

  • DEFAULT_DATE_FORMAT = '%a %d %B %Y' : 默认的日期格式；

  • DISPLAY_PAGES_ON_MENU = True : 是否在导航条显示页面信息；

  • DISPLAY_CATEGORIES_ON_MENU = True : 是否在导航条显示分类；

  • DEFAULT_DATE = None : 获取默认时间，如果设置为'fs'，Pelican就会使用文件系统的时间；

  • DEFAULT_METADATA = {} : 默认的元数据；

  • DOCUTILS_SETTINGS = {} : 这个设置不太懂

  • FILENAME_METADATA = '(?P\d{4}-\d{2}-\d{2}).*' : 根据文件名称设置元数据；

  • PATH_METADATA = '' : 和FILENAME_METADATA类似，基于路径设置元数据；

  • EXTRA_PATH_METADATA = {} :

  • DELETE_OUTPUT_DIRECTORY = False : 是否删除output目录，这是一个危险的设置，谨慎使用；

  • OUTPUT_RETENTION = [] : 如果设置了DELETE_OUTPUT_DIRECTORY，可以通过这个设置防着一些文件被删除；

  • JINJA_EXTENSIONS = [] : Jinjia的扩展目录

  • LOCALE : 修改本地目录，

  • LOG_FILTER = [] : 日志目录；

  • IGNORE_FILES = ['.#*'] : 有些文件是不需要解析输出到output下面的，就可以设置忽略文件，符合对于正则表达式的文件都不会被解析;

  • MD_EXTENSIONS = ['codehilite(css_class=highlight)','extra'] :

  • OUTPUT_PATH = 'output/' : 生成文件的输出目录；

  • PATH : content路径，即存放我们自己编写的文章的路径；

  • PAGE_PATHS = ['pages'] : 页面路径，相对于PATH来说的；

  • PAGE_EXCLUDES = [] : 当Pelican在ARTICLE_PATHS中查找页面的时候，希望排除的目录

  • ARTICLE_PATHS = [''] : 文章路径，相对于content

  • ARTICLE_EXCLUDES = [] : 当Pelican在PAGE_PATHS中查找文章的时候，希望排除的目录

  • OUTPUT_SOURCES = False : 如果设置为True，文章和页面的文件也会不输出到OUTPUT_PATH目录中

  • OUTPUT_SOURCES_EXTENSION = '.text'

URL 设置

首先我们需要理解的是目前URL的两种形式：相对URL和绝对URL，相对URL主要用于本地预览， 绝对URL主要用在发布博客上面，Pelican通过两个不同的配置文件设置开发环境下和发布环境 下的不同的URL模式，使用pelican-quickstart创建博客的时候，会生成两个配置文件， 分别是pelicanconf.py和publishconf.py，当然这个是针对自己有服务器的，如果是 使用GitHub来发布博客的话，pelicanconf.py就够了。

我们可以通过_URL和_SAVE_AS来定制URL，这些配置的值能让我们自己决定我们的文章放在 那个目录下面，在不同的目录下面决定了我们的URL也是不一样的。

温馨提示： 如果希望目录是按时间来设置的，首先使用的是文章的Date属性，如果这个 属性没有设置，将使用文件的mtime属性，可以通过Python的 datetime 模块了解详情。

同时可以使用其他的元数据信息来设置URL

  • slug

  • date

  • lang

  • author

  • category

下面给出几个例子

  • ARTICLE_URL = 'posts/{date:%Y}/{date:%b}/{date:%d}/{slug}/'

  • ARTICLE_SAVE_AS = 'posts/{date:%Y}/{date:%b}/{date:%d}/{slug}/index.html'

  • PAGE_URL = 'pages/{slug}/'

  • PAGE_SAVE_AS = 'pages/{slug}/index.html'

这样发布出的博客文章的链接类似于：/posts/2011/Aug/07/sample-post/index.html， 页面的链接类似于： /pages/about/index.html

和URL配置相关的属性说明如下：

  • ARTICLE_URL = '{slug}.html' : 设置文章的我URL格式；

  • ARTICLE_SAVE_AS = '{slug}.html' : 设置文章在output目录的保存位置；

  • ARTICLE_LANG_URL = '{slug}-{lang}.html' : 如果是多种语言的话，可以通过这个属性进行设置，但是对默认语言是无效的；

  • ARTICLE_LANG_SAVE_AS = '{slug}-{lang}.html' : 一样和多语言设置是相关的；

  • DRAFT_URL = 'drafts/{slug}.html' : 稿件的URL设置；

  • DRAFT_SAVE_AS = 'drafts/{slug}.html' : 稿件在output中保存的位置；

  • DRAFT_LANG_URL = 'drafts/{slug}-{lang}.html' : 不同语言的语言的URL设置，对默认语言无效；

  • DRAFT_LANG_SAVE_AS = 'drafts/{slug}-{lang}.html' : 不同语言的稿件在outp中存放的位置；

  • PAGE_URL = 'pages/{slug}.html' : 页面的URL设置；

  • PAGE_SAVE_AS = 'pages/{slug}.html' : 页面的在output中存放的位置；

  • PAGE_LANG_URL = 'pages/{slug}-{lang}.html' : 不同语言的页面的URL设置，对默认语言无效；

  • PAGE_LANG_SAVE_AS = 'pages/{slug}-{lang}.html' : 不同语言的页面在output中的存放位置；

  • CATEGORY_URL = 'category/{slug}.html' : 分类信息的URL设置；

  • CATEGORY_SAVE_AS = 'category/{slug}.html' : 分类信息的存放位置；

  • TAG_URL = 'tag/{slug}.html' : 标签的URL设置；

  • TAG_SAVE_AS = 'tag/{slug}.html' : 标签信息在output中存放的位置；

  • AUTHOR_URL = 'author/{slug}.html' : 针对作者的URL设置；

  • AUTHOR_SAVE_AS = 'author/{slug}.html' : 作者的存放位置；

  • YEAR_ARCHIVE_SAVE_AS = '' : 按年存放文章的位置；

  • MONTH_ARCHIVE_SAVE_AS = '' : 按月存放文章；

  • DAY_ARCHIVE_SAVE_AS = '' : 按天存放文章；

  • SLUG_SUBSTITUTIONS = () : 这个属性不是太理解；

Pelican可以根据文章的创建的时候，按年月日生成目录，通过这种方式可以让读者清晰的知道这 篇文章是什么时候写的，生成年月日的配置一般都是以_SAVE_AS结尾的

下面是一个样例

  • YEAR_ARCHIVE_SAVE_AS = 'posts/{date:%Y}/index.html'

  • MONTH_ARCHIVE_SAVE_AS = 'posts/{date:%Y}/{date:%b}/index.html'

设置之后我们得到的格式是：posts/2011/index.html和posts/2011/Aug/index.html

日期格式化

如果DATE_FORMATS没有设置，Pelican就会使用DEFAULT_DATE_FORMAT作为默认的日期格式， 如果我们想不同的语言支持不同的日期格式就可以通过设置DATE_FORMATS来实现，日期的详细 格式可以参考Python strftime documentation ，我们可以通过字符-和%来格式化我们想要的日期格式，例如%d/%m/%Y输出01/01/2014， %-d/%-m/%Y输出1/1/2014

DATE_FORMATS = {
    'en': '%a, %d %b %Y',
    'jp': '%Y-%m-%d(%a)',
}

通过LOCALE可以进一步控制日期的格式，不同的操作系统使用不同的日期格式

LOCALE = ('usa', 'jpn',  # On Windows
    'en_US', 'ja_JP'     # On Unix/Linux
    )

如果希望不同的语言，不同的操作系统的日期格式都不同，可以向下面这样设置

# On Unix/Linux
DATE_FORMATS = {
    'en': ('en_US','%a, %d %b %Y'),
    'jp': ('ja_JP','%Y-%m-%d(%a)'),
}

# On Windows
DATE_FORMATS = {
    'en': ('usa','%a, %d %b %Y'),
    'jp': ('jpn','%Y-%m-%d(%a)'),
}
